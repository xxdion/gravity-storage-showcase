SYSTEM EXPLANATION

重力储能智能调度展示平台：前后端说明

这个页面专门解释当前项目到底在做什么。简单说：它是一个可以演示 “场景数据 → 规划策略 → 优化调度 → 控制建议” 的前后端系统原型。 前端负责页面展示和交互，后端负责生成数据、运行演示算法、返回图表和表格需要的结果。

前端是干什么的？

负责“看得见、点得动、展示清楚”
  • 显示登录页、左侧四个模块和右侧工作区。
  • 收集用户输入，比如季节、场景、SOC、权重、预测偏差。
  • 调用后端接口，把结果画成曲线、指标卡、表格。
  • 不再自己计算主要结果，只负责展示后端返回的数据。

后端是干什么的？

负责“算数据、跑逻辑、给结果”
  • 根据输入生成负荷、风电、光伏、电价、净负荷。
  • 根据场景和约束生成规划策略与滚动曲线。
  • 根据目标权重生成 Pareto 解集和推荐调度方案。
  • 根据预测偏差和调度结果生成下一周期控制建议。

整个系统流程

01

多场景建模

输入场景、季节、设备参数，输出源荷曲线和场景摘要。

02

规划策略

输入容量、启停、滚动周期，输出中长期/短期/日内策略。

03

优化调度

输入 SOC 和目标权重，输出 Pareto、推荐方案和调度表。

04

实时控制建议

输入预测模型和实时偏差，输出下一周期充放电建议。

前后端怎么连接？

现在是真实的吗？

现在已经是“真实前后端分离”。页面不是自己算结果，而是调用后端接口拿结果。 但是算法本身还是第一版演示算法，不是完整真实 NSGA-II，也没有接真实现场设备。

  • 真实 前端调用后端接口。
  • 真实 后端根据输入返回不同数据。
  • 模拟 负荷、风电、光伏、电价样例数据。
  • 模拟 优化调度和预测控制算法。

为什么这样做？

这样做的好处是：会议上能演示一个完整系统，后续也能平滑升级。 将来有真实数据或真实算法时，只需要替换后端算法函数，前端页面不用大改。

当前版本适合讲：系统具备“建模、规划、优化、控制建议”的完整软件形态。 后续版本再讲：接入真实数据、真实 NSGA-II、真实预测模型。

接口说明

接口 输入 输出 页面对应模块
GET /api/health 无 后端状态、算法版本、时间 顶部后端连接状态
GET /api/bootstrap 无 默认参数、默认曲线、默认优化结果 页面初始化
POST /api/scenario/simulate 场景、季节、建筑类型、设备参数 源荷曲线、模型指标、场景摘要 多场景重力储能模型
POST /api/planning/run 场景参数、规划参数 规划策略、日前曲线、日内滚动曲线 多时间尺度规划策略
POST /api/optimization/run 场景、规划、SOC、权重、迭代参数 Pareto、推荐方案、调度表、SOC 曲线 智能优化调度算法
POST /api/control/preview 场景、推荐方案、预测偏差、控制模式 预测曲线、下一周期控制建议、控制表 负荷预测实时控制

一句话讲给别人听

这个系统目前是一套重力储能智能调度展示原型：前端负责交互和可视化， 后端负责生成源荷数据、规划策略、优化调度和控制建议。当前算法是可解释演示算法， 后续可以替换为真实 NSGA-II、真实预测模型和现场数据接口。
